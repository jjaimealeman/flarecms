**1. Add `rollbackMigration` to SchemaMigrationService:**

```typescript
async rollbackMigration(migrationId: string, userId: string | null): Promise<{ success: boolean, error?: string }>
```

Logic:
- Fetch the migration by ID from schema_migrations table
- Validate: must have `status === 'applied'` (cannot rollback already rolled-back migration)
- Validate: must be the MOST RECENT migration for its collection (query: `SELECT id FROM schema_migrations WHERE collection_id = ? AND status = 'applied' ORDER BY applied_at DESC LIMIT 1` -- if the result ID doesn't match migrationId, return error "Can only rollback the most recent migration")
- Parse `previous_schema` from the migration record
- If `previous_schema` is null, return error "No previous schema available for rollback"
- Use `db.batch()` to atomically:
  1. Update the collection's schema to the previous_schema JSON
  2. Update the collection's updated_at timestamp
  3. Update the migration's status to 'rolled_back', set rolled_back_at and rolled_back_by
- Record a NEW migration entry with type 'rollback' that captures the schema BEFORE rollback (so rollback itself is auditable)
- Return `{ success: true }`

On any error, return `{ success: false, error: message }`.

**2. Add POST `/rollback/:id` endpoint to admin-schema-migrations.ts:**

- Requires auth (already applied via middleware)
- Get migration ID from params
- Create `SchemaMigrationService(db)` and call `rollbackMigration(id, user.userId)`
- If HTMX request (`HX-Request` header):
  - On success: return HTML fragment with success alert and `HX-Redirect: /admin/schema-migrations` header to refresh the page
  - On error: return HTML fragment with error alert
- If regular request:
  - On success: redirect to /admin/schema-migrations
  - On error: redirect back with error query param

**3. Add rollback button to migration history template:**

In the migration history template (Plan 03's template), add a "Rollback" button on migration entries that are:
- `status === 'applied'`
- The MOST RECENT migration for their collection

The button should:
- Use HTMX: `hx-post="/admin/schema-migrations/rollback/{id}"` with `hx-confirm="Are you sure you want to rollback this migration? This will restore the previous schema."`
- Be styled as a small danger button (red/rose theme)
- Show "Rollback" text with an undo icon if available

Migrations with status 'rolled_back' or 'failed' should NOT show the rollback button.
Non-latest migrations should show a disabled state or no button (with tooltip "Only the most recent migration can be rolled back").
