  ```
  cd /home/jaime/www/_github/sonicjs/my-astro-cms
  npx wrangler d1 execute DB --local --command "CREATE TABLE IF NOT EXISTS user_collection_permissions (id TEXT PRIMARY KEY, user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE, collection_id TEXT NOT NULL REFERENCES collections(id) ON DELETE CASCADE, role TEXT NOT NULL, granted_by TEXT NOT NULL REFERENCES users(id), created_at INTEGER NOT NULL, UNIQUE(user_id, collection_id));"
  npx wrangler d1 execute DB --local --command "CREATE INDEX IF NOT EXISTS idx_ucp_user ON user_collection_permissions(user_id);"
  npx wrangler d1 execute DB --local --command "CREATE INDEX IF NOT EXISTS idx_ucp_collection ON user_collection_permissions(collection_id);"
  ```

  **Part B: Create RBAC service**

  Create `sonicjs-fork/packages/core/src/services/rbac.ts` with:

  1. `checkCollectionPermission(db: D1Database, userId: string, globalRole: string, collectionId: string, requiredAction: 'read' | 'create' | 'edit' | 'publish'): Promise<boolean>`
     - Admin bypasses all checks: `if (globalRole === 'admin') return true`
     - Query `user_collection_permissions` for user + collection
     - No permission record = no access (return false)
     - Viewer: only 'read' allowed
     - Editor: all actions (read, create, edit, publish)
     - Author: read, create allowed; edit allowed ONLY for own content (return true for edit but caller must also check `content.author_id === userId`)

  2. `getCollectionPermissions(db: D1Database, userId: string): Promise<Array<{ collectionId: string, role: string }>>`
     - Returns all collection permissions for a user
     - Used by admin UI to show which collections user has access to

  3. `grantCollectionPermission(db: D1Database, params: { userId: string, collectionId: string, role: string, grantedBy: string }): Promise<void>`
     - INSERT OR REPLACE into `user_collection_permissions`
     - Validates role is one of: 'editor', 'author', 'viewer'
     - Admin role is never granted at collection level (admin is always global)

  4. `revokeCollectionPermission(db: D1Database, userId: string, collectionId: string): Promise<void>`
     - DELETE from `user_collection_permissions`

  Export all functions. Add to core package `index.ts`.

  <action>
  **Part A: Enforce in API content CRUD (api-content-crud.ts)**

  Import `checkCollectionPermission` from rbac service.

  1. POST /api/content (create):
     - After auth check, get `collectionId` from body
     - Call `checkCollectionPermission(db, user.userId, user.role, collectionId, 'create')`
     - If false, return `c.json({ error: 'Insufficient permissions for this collection' }, 403)`

  2. PUT /api/content/:id (update):
     - After loading existing content, call `checkCollectionPermission(db, user.userId, user.role, existing.collection_id, 'edit')`
     - If false, return 403
     - If user's collection role is 'author', additionally check: `if (existing.author_id !== user.userId) return 403`

  3. DELETE /api/content/:id (delete):
     - Same as edit check (collection edit permission required for delete)

  4. GET routes should NOT enforce RBAC — public read access is the default. API tokens (Plan 07) handle read scoping separately.

  **Part B: Enforce in admin content routes (admin-content.ts)**

  The admin is server-rendered HTMX. For every mutating route (POST/PUT that creates or updates content):
  - Check collection permission before processing
  - On failure, render an HTMX error fragment: `<div class="alert alert-error">You don't have permission to edit content in this collection</div>`

  For the content LIST page:
  - If user is NOT admin, filter the collections shown to only those the user has permissions for
  - Use `getCollectionPermissions()` to get user's allowed collections

  **Part C: Protect admin-users.ts routes**

  Admin user management routes (create user, edit user, assign roles) must require admin role:
  - Add `requireAuth()` and check `user.role === 'admin'` at the top of user management handlers
  - Non-admin users trying to access user management get redirected or shown error

  After changes: `cd /home/jaime/www/_github/sonicjs/sonicjs-fork && pnpm build && cd /home/jaime/www/_github/sonicjs/my-astro-cms && pnpm install`
