<task type="auto">
  <name>Task 1: Create SearchModal component with client-side search</name>
  <files>packages/site/src/components/SearchModal.astro</files>
  <action>
    Create `packages/site/src/components/SearchModal.astro` with two parts: HTML structure and a bundled script.

    **HTML structure (server-rendered):**
    - Hidden overlay container (`id="search-modal"`, `class="hidden"`) with:
      - Backdrop div (semi-transparent, click-to-close)
      - Modal panel: On desktop (`lg:` breakpoint) -- dropdown positioned below the header, max-w-2xl, centered. On mobile -- full-screen fixed overlay.
      - Input field at top with search icon, placeholder "Search docs...", autofocus when opened
      - Results container below input
      - Footer with keyboard hints: arrows to navigate, Enter to select, Esc to close

    **Client-side script (`<script>` WITHOUT `is:inline` -- Astro bundles MiniSearch with it):**

    This is the FIRST script in the codebase that uses module bundling (not `is:inline`). This is necessary because MiniSearch needs to be imported as an npm module.

    1. **Imports:** `MiniSearch` from 'minisearch', `MINISEARCH_OPTIONS` and `SearchDocument` type from '../lib/search-config'

    2. **State:** `searchIndex: MiniSearch<SearchDocument> | null`, `indexLoading: boolean`, `selectedIndex: number`

    3. **Lazy index loading:** `async function loadIndex()` fetches `/api/search-index.json`, calls `MiniSearch.loadJSON<SearchDocument>(json, MINISEARCH_OPTIONS)`, caches in `searchIndex` variable. Shows "Loading..." state in results area during fetch. On error, shows "Failed to load search index" message.

    4. **Open/close:** `openSearch()` removes hidden class from modal, focuses input, calls `loadIndex()` if not loaded. `closeSearch()` adds hidden class, clears input and results, resets selectedIndex. Use `inert` attribute on `<main>` when modal is open to trap focus.

    5. **Global keyboard listener:** Cmd+K / Ctrl+K calls `e.preventDefault()` + `openSearch()`. Escape calls `closeSearch()`. Only when modal is open: ArrowDown/ArrowUp navigate results (update `selectedIndex`, add `data-search-active` attribute, scrollIntoView nearest), Enter navigates to the active result's link.

    6. **Search execution with debounce:** On input event, debounce at 150ms. If query length < 2, show empty state (popular docs links). Otherwise call `searchIndex.search(query, { limit: 10 })` and render results.

    7. **Result rendering:** Each result as a clickable `<a>` element containing:
       - Section badge (small pill with section name, e.g., "Getting Started", "API Reference")
       - Page title with match highlighting (use `<mark>` tags with `bg-flare-500/30` style)
       - If headingText exists, show it as sub-text: "Section heading: {headingText}" also highlighted
       - Build href: For docs results, `/docs/${sectionSlug}/${slug}` + optional `#${headingId}`. For blog: `/blog/${slug}`. For news: `/news/${slug}`.

    8. **Match highlighting:** `highlightMatches(text, terms)` escapes HTML first, then wraps matched terms in `<mark class="bg-flare-500/30 text-flare-300 rounded px-0.5">`. Get matched terms from `result.match` property of MiniSearch results.

    9. **Empty/no-results states:**
       - Before typing: Show curated list of 5-6 popular docs links (Getting Started, Installation, API Reference, Collections, Deployment, Configuration) as clickable items
       - After typing with no results: "No results for '{query}'" message + the same popular links

    10. **Debounce helper:** Inline 5-line debounce function (from research doc).

    **Styling notes:**
    - Dark theme matching site: bg-slate-800 modal, bg-slate-900 input, slate-300 text
    - Flare orange accents for active result (border-l-2 border-flare-500 or bg-flare-500/10)
    - Results scroll area with max-h so modal doesn't grow beyond viewport
    - Smooth transitions on open/close (opacity + transform)
    - Platform-aware Cmd/Ctrl badge: detect Mac via navigator.platform and show appropriate text
  </action>
