    **Seed Script (`seed-docs.ts`):**

    Build a TypeScript script that reads markdown files from `content/docs/` and pushes them to the CMS API. Follow the pattern from `seed-admin.ts` for the `--direct` D1 mode using `getPlatformProxy()`.

    Script interface:
    ```bash
    npx tsx scripts/seed-docs.ts http://localhost:8787          # API mode (default)
    npx tsx scripts/seed-docs.ts https://flare-cms.jjaimealeman.workers.dev  # Production
    npx tsx scripts/seed-docs.ts --direct                       # Direct D1 mode
    ```

    API mode flow:
    1. Parse CLI args for base URL
    2. Authenticate via POST `/auth/login` with form-encoded body (`application/x-www-form-urlencoded`). Use `URLSearchParams`. Credentials from env vars `FLARE_ADMIN_EMAIL` / `FLARE_ADMIN_PASSWORD` with fallback to `jjaimealeman@gmail.com` / empty string. Handle `redirect: 'manual'` and extract JWT from `token=` cookie in `Set-Cookie` header.
    3. Get collection IDs: GET `/api/collections`, find `docs` and `docs-sections` by name, extract their `id` fields.
    4. Wipe existing content (idempotent): List all docs pages via GET `/api/collections/docs/content?limit=1000`, delete each via DELETE `/api/content/{id}`. Then list all sections via GET `/api/collections/docs-sections/content?limit=1000`, delete each. ORDER MATTERS: pages first (FK to sections), then sections.
    5. Read all `_section.md` files using glob pattern `content/docs/*/_section.md`. Parse with gray-matter. Create each section via POST `/api/content` with: `collectionId` (docs-sections UUID), `title` (= name), `slug`, `status: 'published'`, `data: { name, slug, description, order }`. Capture returned `data.id` as section UUID. Store slug-to-UUID mapping.
    6. Read all non-`_section.md` markdown files using glob pattern `content/docs/**/*.md` excluding `_section.md`. Parse with gray-matter. For each file: determine section from parent directory name, look up section UUID from the mapping. Create page via POST `/api/content` with: `collectionId` (docs UUID), `title`, `slug`, `status: 'published'`, `data: { title, slug, excerpt, content (raw markdown body after frontmatter), section (UUID), order, status: 'published', lastUpdated }`.
    7. Normalize `\r\n` to `\n` in content before POSTing.
    8. Print summary: sections created, pages created, any errors.

    Direct D1 mode (`--direct` flag):
    - Use `getPlatformProxy()` like `seed-admin.ts`
    - Use `createDb(env.DB)` from `@flare-cms/core`
    - Import `content` and `collections` tables from `@flare-cms/core`
    - Do the same wipe-and-reseed but with direct Drizzle ORM inserts
    - Generate UUIDs with `crypto.randomUUID()`

    Important details:
    - All fetch calls use `Cookie: token=${jwt}` header (not Authorization Bearer)
    - Content-Type for API calls is `application/json`
    - Auth endpoint is `application/x-www-form-urlencoded`
    - Add helpful console.log for each step (with counts)
    - Handle errors gracefully — log and continue, don't crash on a single page failure

    **Prompt Generator (`generate-prompt.ts`):**

    Build a TypeScript script that reads the live CMS schema and outputs a self-contained `PROMPT.md` file.

    Script interface:
    ```bash
    npx tsx scripts/generate-prompt.ts http://localhost:8787    # Reads from CMS API
    ```

    Flow:
    1. GET `/api/collections` to fetch all collection schemas
    2. Build a markdown document containing:
       - System context: What Flare CMS is, how the API works
       - For each collection: name, display name, all fields with types and constraints
       - API endpoint reference (GET/POST/PUT/DELETE with paths)
       - Example request/response pairs for creating content
       - Auth instructions (form-encoded login, JWT cookie)
       - Instructions for an LLM to generate seed content
    3. Write output to `PROMPT.md` in the current directory (or stdout if piped)

    The prompt generator does NOT need auth — collections endpoint is public. Keep it simple.
  </action>
