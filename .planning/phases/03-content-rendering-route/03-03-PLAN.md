<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Complete content rendering pipeline: Shiki syntax highlighting, copy buttons, callout boxes, tabbed code examples, heading anchors, TOC with scroll-spy, external link handling, and image lightbox.</what-built>
  <how-to-verify>
Visit a docs page at http://localhost:4321/docs/{section}/{page} that has code blocks, and check:

1. **Syntax highlighting** (RENDER-01): Code blocks have colored syntax (catppuccin-mocha dark theme). Language label visible in top-right corner of each block.

2. **Copy button** (RENDER-02): Hover over a code block — "Copy" button appears top-right. Click it — text changes to "Copied!" for 2 seconds. Paste into a text editor — should be plain code, no HTML tags.

3. **Callouts** (RENDER-03): If a doc has `> [!NOTE] text` or similar — should render as a styled box with icon. If no docs have callouts yet, create a quick test doc in the CMS with:
   ```
   > [!NOTE]
   > This is a note callout.

   > [!WARNING]
   > This is a warning.

   > [!TIP]
   > This is a tip.

   > [!CAUTION]
   > This is dangerous.
   ```

4. **Heading anchors**: Hover over any h2 or h3 — a "#" link should appear. Clicking it updates the URL hash.

5. **Table of Contents**: Right sidebar (visible at xl+ widths) shows "On this page" with h2/h3 headings. Scroll down — the active heading highlights in the TOC.

6. **Lightbox**: Click any image in a docs page — should open in an overlay. Click overlay or press Escape to close.

7. **External links**: Any external link should have a small indicator icon and open in a new tab.
  </how-to-verify>
