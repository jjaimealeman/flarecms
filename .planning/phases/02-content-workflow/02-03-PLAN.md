  **Part A: Schema migration**

  Run against local D1 to add the two new columns to the existing `workflow_history` table (migration 001 schema):
  ```
  cd /home/jaime/www/_github/sonicjs/my-astro-cms
  npx wrangler d1 execute DB --local --command "ALTER TABLE workflow_history ADD COLUMN changed_fields TEXT;"
  npx wrangler d1 execute DB --local --command "ALTER TABLE workflow_history ADD COLUMN action_type TEXT DEFAULT 'status_change';"
  ```

  If ALTER TABLE fails because column already exists, that's fine — `IF NOT EXISTS` is not supported for ALTER TABLE ADD COLUMN in SQLite, so just catch/ignore the error.

  **Part B: Create audit trail service**

  Create `sonicjs-fork/packages/core/src/services/audit-trail.ts`:

  1. `logStatusChange(db: D1Database, params: { contentId: string, fromStatus: string, toStatus: string, userId: string, comment?: string }): Promise<void>`
     - Inserts into `workflow_history` with `action_type = 'status_change'`
     - Uses `crypto.randomUUID()` for id, `Date.now()` for `created_at`
     - `changed_fields` is null for status-only changes

  2. `logContentEdit(db: D1Database, params: { contentId: string, status: string, userId: string, changedFields: Record<string, { old: any, new: any }> }): Promise<void>`
     - Inserts into `workflow_history` with `action_type = 'content_edit'`
     - `from_status` and `to_status` both set to current status (no transition)
     - `action = 'content_edit'`
     - `changed_fields` = JSON.stringify of the diff object

  3. `computeFieldDiff(existing: Record<string, any>, updated: Record<string, any>, fields: string[]): Record<string, { old: any, new: any }>` helper
     - For each field in `fields`, compare `existing[field]` vs `updated[field]`
     - Only include fields that actually changed
     - Return empty object if nothing changed

  Export all functions. Add export to `sonicjs-fork/packages/core/src/index.ts`.
  </action>

  2. Verify columns exist: `cd /home/jaime/www/_github/sonicjs/my-astro-cms && npx wrangler d1 execute DB --local --command "PRAGMA table_info(workflow_history);"`
  </verify>

  **Part A: Wire into PUT /api/content/:id (api-content-crud.ts)**

  Import `{ logStatusChange, logContentEdit, computeFieldDiff }` from audit-trail service.

  In the PUT handler, AFTER the successful UPDATE query but BEFORE returning the response:

  1. If status changed (body.status !== existing.status):
     - Call `logStatusChange(db, { contentId: id, fromStatus: existing.status, toStatus: body.status, userId: user.userId })`

  2. If content fields changed (title, slug, data):
     - Call `computeFieldDiff(existing, body, ['title', 'slug', 'data'])` to get changed fields
     - If any fields changed, call `logContentEdit(db, { contentId: id, status: existing.status, userId: user.userId, changedFields })`

  Wrap audit calls in try/catch — audit failure should NOT block the content update (log error but return success).

  **Part B: Wire into admin content routes (admin-content.ts)**

  Find the admin POST/PUT handlers that update content. Add the same audit logging:
  - On status change: `logStatusChange()`
  - On content edit: `logContentEdit()`
  - Same try/catch pattern — audit failure is non-blocking
