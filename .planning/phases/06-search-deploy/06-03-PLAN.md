  <name>Task 1: Add production safety flag to seed script</name>
  <files>packages/cms/scripts/seed-docs.ts</files>
  <action>
    Update `packages/cms/scripts/seed-docs.ts` to add production URL detection and safety confirmation:

    1. After parsing the base URL from argv, check if it's a non-localhost URL (i.e., does NOT match `localhost`, `127.0.0.1`, or `0.0.0.0`).

    2. If the URL is non-localhost:
       a. Check if `--confirm-production` flag is present in `process.argv`
       b. If NOT present, print a clear warning:
          ```
          WARNING: You are about to seed against a production URL:
            ${baseUrl}

          This will DELETE all existing docs content and recreate it.

          To confirm, re-run with --confirm-production flag:
            npx tsx scripts/seed-docs.ts ${baseUrl} --confirm-production
          ```
          Then `process.exit(1)`.
       c. If `--confirm-production` IS present, print:
          ```
          Production mode confirmed. Target: ${baseUrl}
          ```
          and continue execution.

    3. For localhost URLs, no flag is needed -- proceed normally.

    4. Place this check early in the main function, before authentication, so it fails fast.
  </action>
  <verify>
    Run `cd packages/cms && npx tsx scripts/seed-docs.ts https://flare-cms.jjaimealeman.workers.dev` (without flag) -- should print warning and exit with code 1. Run `npx tsx scripts/seed-docs.ts http://localhost:8787` -- should proceed normally (will fail at auth if server not running, which is fine for verification).
  </verify>
  <done>Seed script exits with clear warning when targeting non-localhost URLs without --confirm-production flag. Localhost URLs work without the flag.</done>
