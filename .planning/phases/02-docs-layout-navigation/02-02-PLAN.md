    **Step 2: Create docs/index.astro (landing page at /docs)**

    This page lists all documentation sections with descriptions, driven by CMS data. It serves as the breadcrumb target for "Docs".

    - Import `Layout` from `../../layouts/Layout.astro`
    - Import `getDocsSections, getDocsPages` from `../../lib/flare`
    - Import `buildNavTree` from `../../lib/docs-nav`

    - Show a heading "Documentation" with subtitle
    - Grid of section cards (similar style to existing docs.astro placeholder but driven by CMS data)
    - Each card shows section name, description (from section.data.description), page count, and links to the first page in that section (`section.pages[0]?.href || '#'`)
    - Use section icon from the same Lucide icon map strategy as DocsSidebar (import the same icons)
    - Style: rounded-xl border border-slate-800 bg-slate-800/30 hover:bg-slate-800/60 hover:border-slate-700 (matching existing card style)

    **Step 3: Create docs/[...slug].astro (catch-all route)**

    This is the core docs page route. It handles `/docs/section-slug/page-slug`.

    - This is SSR mode — do NOT use `getStaticPaths()`. Use `Astro.params.slug` directly.
    - Import `DocsLayout` from `../../layouts/DocsLayout.astro`
    - Import `getDocsSections, getDocsPages` from `../../lib/flare`
    - Import `buildNavTree, flattenForPrevNext` from `../../lib/docs-nav`

    In the frontmatter:
    1. Parse `Astro.params.slug` — split by `/` to get `[sectionSlug, pageSlug]`
    2. If slug is undefined, empty, or doesn't have exactly 2 parts, return `Astro.redirect('/docs')` (graceful fallback)
    3. Fetch sections and docs with `Promise.all([getDocsSections(), getDocsPages()])`
    4. Build nav tree with `buildNavTree(sections, docs)`
    5. Find current section in navSections by matching `section.slug === sectionSlug`
    6. Find current page in that section by matching `page.slug === pageSlug`
    7. If section or page not found, return a 404 response: `return new Response(null, { status: 404, statusText: 'Not Found' })`
    8. Flatten for prev/next: `flattenForPrevNext(navSections)`
    9. Find currentIndex in flattened array by matching `page.href === currentPath`
    10. Build breadcrumbs: `[{ label: 'Docs', href: '/docs' }, { label: currentSection.name, href: currentSection.pages[0]?.href || '/docs' }, { label: currentPage.title, href: currentPath }]`
    11. Build prev/next: prev = allPages[currentIndex - 1] with sectionName, next = allPages[currentIndex + 1] with sectionName. Map to `{ title, section: sectionName, href }`.
    12. Find the full doc object from the original `docs` array to get `doc.data.content` for rendering

    In the template:
    - Wrap everything in `<DocsLayout>` passing all computed props (title, section, breadcrumbs, prev, next, sections: navSections, currentPath)
    - Inside the default slot: render `doc.data.content` using `<Fragment set:html={doc.data.content} />` (raw HTML from CMS markdown — Phase 3 adds Shiki/rehype processing)
    - Leave the `toc` named slot empty for now (Phase 3)

    **Important SSR notes:**
    - Use `const currentPath = Astro.url.pathname` — normalize trailing slash if needed
    - The `doc.data.section` field is a content ID string, NOT a slug. The nav tree builder already handles the join. To find the original doc for content, match by: find the doc in `docs` array where `doc.data.slug === pageSlug` AND the doc belongs to the current section (doc.data.section matches the section's id from the sections array).
