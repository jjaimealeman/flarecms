**Wire plugins in createSonicJSApp()** — add plugin installation AFTER the bootstrap middleware line (line 177: `app.use('*', bootstrapMiddleware(config, app))`) but BEFORE custom middleware (line 180). This is important because:
- Bootstrap must run first (it sets up DB, migrations, app reference)
- Plugins register before custom middleware so their middleware can participate in the request chain

Add this block after line 177:

```typescript
// Install user-registered plugins
if (config.plugins?.instances?.length && !config.plugins?.disableAll) {
  const { PluginManager: PluginManagerImpl } = await import('./plugins/plugin-manager')
  const { getAppReference } = await import('./middleware/bootstrap')

  // Use a one-time middleware to install plugins after first request
  // (bootstrap must run first to initialize DB bindings)
  let pluginsInstalled = false
  app.use('*', async (c, next) => {
    if (!pluginsInstalled) {
      pluginsInstalled = true
      const pm = new PluginManagerImpl()
      await pm.initialize({
        db: c.env.DB,
        kv: c.env.CACHE_KV,
        r2: c.env.MEDIA_BUCKET,
        config: { enabled: true },
        services: {} as any,  // Services not available at boot — plugins use hooks for service interaction
        hooks: pm.hooks,
        logger: { debug: console.debug, info: console.info, warn: console.warn, error: console.error },
        app
      })
      for (const plugin of config.plugins!.instances!) {
        await pm.install(plugin)
      }
    }
    await next()
  })
}
```

**IMPORTANT implementation notes:**
- Use dynamic `await import()` for PluginManager to avoid circular imports (app.ts imports routes which may import hook-system)
- The `pluginsInstalled` module-level flag follows the same pattern as `kvInitialized` (line 133) — runs once per isolate
- `services: {} as any` is acceptable because plugins access services through hooks, not direct service references. The PluginContext.services field is typed but optional in practice.
- Pass `app` directly (the Hono instance) so plugins can call `app.use()` or `app.route()` in their activate() callback

**Alternative (simpler, preferred if no circular import issue):** Use a static import instead of dynamic import. Check whether importing PluginManager at the top of app.ts causes any circular dependency. If it does NOT, use static import. If it DOES, use the dynamic import pattern above.

To test for circular imports: add `import { PluginManager as PluginManagerImpl } from './plugins/plugin-manager'` at top of app.ts and run `pnpm build`. If it succeeds, use static import and simplify the middleware to a synchronous check.
