  <action>
Refactor both upload code paths (API single upload, API bulk upload, admin upload) to use `file.stream()` for non-image files and keep `file.arrayBuffer()` for images only (because `getImageDimensions()` needs the ArrayBuffer).

**In api-media.ts — single upload route (line ~80):**
Replace:
```typescript
const arrayBuffer = await file.arrayBuffer()
const uploadResult = await c.env.MEDIA_BUCKET.put(r2Key, arrayBuffer, { ... })
```
With:
```typescript
let uploadBody: ArrayBuffer | ReadableStream
let arrayBuffer: ArrayBuffer | null = null

if (file.type.startsWith('image/') && !file.type.includes('svg')) {
  // Buffer images for dimension extraction (images are typically < 10MB)
  arrayBuffer = await file.arrayBuffer()
  uploadBody = arrayBuffer
} else {
  // Stream non-image files directly to R2 — avoids 128MB Worker memory limit
  uploadBody = file.stream()
}

const uploadResult = await c.env.MEDIA_BUCKET.put(r2Key, uploadBody, { ... })
```
Then update the dimension extraction block (line ~105) to use the local `arrayBuffer` variable — it's already conditional on `file.type.startsWith('image/')` so it will only run when arrayBuffer is populated.

**In api-media.ts — bulk upload route (line ~235):**

**In admin-media.ts — admin upload route (line ~477):**
Apply the same pattern. The arrayBuffer at line 477 becomes conditional. The dimension extraction at line 504 already guards on image type.

**Important:** Do NOT touch `getImageDimensions()` function itself — it takes ArrayBuffer and returns {width, height}. That stays the same.
