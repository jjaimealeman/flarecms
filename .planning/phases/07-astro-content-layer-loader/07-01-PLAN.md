<task type="auto">
  <name>Task 2: Implement schema converter and API client</name>
  <files>packages/astro/src/schema.ts, packages/astro/src/client.ts, packages/astro/src/index.ts</files>
  <action>
    Create `packages/astro/src/schema.ts`:

    1. Import `z` from `astro/zod` and `CollectionSchema`, `FieldConfig` types from `@flare-cms/core`
    2. Create `FIELD_TYPE_MAP: Record<string, () => z.ZodTypeAny>` mapping all Flare field types to Zod constructors:
       - `string` -> `z.string()`
       - `number` -> `z.number()`
       - `boolean` -> `z.boolean()`
       - `date` -> `z.coerce.date()`
       - `datetime` -> `z.coerce.date()`
       - `email` -> `z.string().email()`
       - `url` -> `z.string().url()`
       - `richtext` -> `z.string()`
       - `markdown` -> `z.string()`
       - `quill` -> `z.string()`
       - `tinymce` -> `z.string()`
       - `mdxeditor` -> `z.string()`
       - `textarea` -> `z.string()`
       - `json` -> `z.unknown()`
       - `select` -> `z.string()` (refined with enum if available)
       - `multiselect` -> `z.array(z.string())`
       - `checkbox` -> `z.boolean()`
       - `radio` -> `z.string()`
       - `slug` -> `z.string()`
       - `color` -> `z.string()`
       - `media` -> `z.string()`
       - `file` -> `z.string()`
       - `reference` -> `z.string()`
       - `array` -> `z.array(z.unknown())`
       - `object` -> `z.record(z.unknown())`
    3. Export `flareSchemaToZod(schema: CollectionSchema): z.ZodObject<any>`:
       - Initialize empty shape `Record<string, z.ZodTypeAny>`
       - For each `[key, field]` in `schema.properties`:
         - Get base Zod field from `FIELD_TYPE_MAP[field.type]?.()` or fall back to `z.string()`
         - If `field.type === 'select'` and `field.enum?.length`, use `z.enum(field.enum as [string, ...string[]])`
         - If key is NOT in `schema.required` array, wrap with `.optional().nullable()` (CMS may return null for empty fields)
       - Add system fields to shape:
         - `_status: z.string().optional()`
         - `_createdAt: z.coerce.date().optional()`
         - `_updatedAt: z.coerce.date().optional()`
       - Return `z.object(shape)`

    Create `packages/astro/src/client.ts`:

    1. Import `FlareLoaderOptions`, `FlareContentItem`, `FlareApiResponse` from `./types`
    2. Export class `FlareClient`:
       - Constructor takes `options: Pick<FlareLoaderOptions, 'apiUrl' | 'apiToken'>` and stores them
       - Private method `headers(): Record<string, string>` returning `{ 'Content-Type': 'application/json' }` plus `{ 'X-API-Key': this.options.apiToken }` if apiToken is set
       - `async fetchCollection(collection: string): Promise<FlareContentItem[]>` — fetches `${apiUrl}/api/collections/${collection}/content`, returns `result.data`
       - `async fetchCollectionSchema(collection: string): Promise<CollectionSchema | null>` — fetches `${apiUrl}/api/collections`, finds collection by name, returns its `schema` property. Import `CollectionSchema` type from `@flare-cms/core`.
       - `async fetchItem(collection: string, id: string): Promise<FlareContentItem | null>` — fetches `${apiUrl}/api/collections/${collection}/content/${id}`, returns `result.data` or null on error
       - All fetch methods include try/catch, returning empty arrays/null on network errors (not crashing builds)

    Create `packages/astro/src/index.ts` as barrel export:
    ```typescript
    export { flareSchemaToZod } from './schema'
    export { FlareClient } from './client'
    export type { FlareLoaderOptions, FlareContentItem, FlareApiResponse } from './types'
    ```

    Note: The `flareLoader` and `flareLiveLoader` exports will be added in Plans 02 and 03.
