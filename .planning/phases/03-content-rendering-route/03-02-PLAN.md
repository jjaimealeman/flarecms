**Update `packages/site/src/layouts/DocsLayout.astro`:**

Add the `headings` prop to the Props interface:
```typescript
headings?: Array<{ id: string, text: string, level: number }>
```
Destructure it from Astro.props (default to empty array).

After the existing mobile menu `<script is:inline>` block, add four more `<script is:inline>` blocks. Use `is:inline` so the scripts run on every page load (no Astro bundling/deduplication which would break on navigation).

**Script 1: Copy Button**
Follow the pattern from 03-RESEARCH.md "Copy Button (Vanilla JS)":
- Query all `[data-rehype-pretty-code-figure]` elements
- For each, find the `pre` and `code` child elements
- Create a button with class `copy-btn`, aria-label "Copy code", text "Copy"
- On click: `navigator.clipboard.writeText(code.textContent || '')`, change text to "Copied!", add `.copied` class, reset after 2000ms
- Set `pre.style.position = 'relative'` and append button to pre

**Script 2: Tab Groups**
Follow the pattern from 03-RESEARCH.md "Tab System (Vanilla JS)":
- Query all `[data-tab-group]` elements
- For each group: get all `[data-tab]` buttons and `[data-tab-panel]` panels
- Get category from `data-tab-group` attribute
- Restore preference from localStorage key `tabPreferences` (JSON object)
- On tab click: deactivate all tabs/panels in group, activate clicked tab + matching panel, persist preference, sync other groups with same category on the page
- On init: activate preferred tab or first tab

**Script 3: Scroll-Spy for TOC**
Follow the pattern from 03-RESEARCH.md "Scroll-Spy with IntersectionObserver":
- Query all `h2[id], h3[id]` headings in the content area
- Query all `[data-toc-link]` elements in the TOC
- Create IntersectionObserver with `rootMargin: '-80px 0px -80% 0px'`
- On intersection: remove active classes from all TOC links, add `text-flare-400 font-medium` to the matching `[data-toc-link="{id}"]`
- Observe all headings

**Script 4: Lightbox**
Simple vanilla JS:
- Query all images inside `.prose` container
- For each image: set `cursor: pointer`, add click listener
- On click: create an overlay div (fixed inset-0, z-[60], bg-slate-950/90, flex items-center justify-center, cursor-pointer)
- Create an img element inside the overlay with the same src, max-w-[90vw] max-h-[90vh] object-contain rounded-lg
- Append overlay to body
- On overlay click or Escape key: remove overlay

IMPORTANT: Keep all existing DocsLayout content intact. Only add the headings prop and the four script blocks at the end (before the closing `</Layout>` tag, after the mobile menu script).
