1. **Add GET route in admin-preview.ts:**

Add a new GET route: `GET /:collectionId/:contentId` (auth-protected) that:
- Gets the collection via `getCollection(db, collectionId)`
- Gets collection fields via `getCollectionFields(db, collectionId)`
- Gets existing content via the content service (if contentId is not 'new')
- Determines the site URL from `c.env.SITE_URL` or `c.env.CORS_ORIGINS` (parse first origin) or default to `http://localhost:4321`
- Calls `renderAdminPreviewPage({ collection, content, fields, siteUrl, apiUrl })` and returns the HTML
- The apiUrl should be the CMS origin (for client-side JS to call /api/preview/draft)

This route needs auth (requireAuth) because only CMS admins should access the preview page. Mount it so that /admin/preview/:collectionId/:contentId serves the page.

Update the route registration in app.ts if needed -- the route may need to be mounted at `/admin/preview` in addition to `/api/preview`. If Plan 01 only mounted at `/api/preview`, add a second mount: `app.route('/admin/preview', adminPreviewRoutes)`. The GET /:collectionId/:contentId route serves the page, the POST /draft and GET /draft/:token serve the API.

Actually, cleaner approach: export TWO route instances from admin-preview.ts:
- `adminPreviewApiRoutes` -- the draft API (POST /draft, GET /draft/:token) -- mounted at /api/preview
- `adminPreviewPageRoutes` -- the page route (GET /:collectionId/:contentId) -- mounted at /admin/preview

Or use a single router and let the mount points handle it. The simplest: keep one router, mount at `/admin/preview` for the page route. The draft API POST/GET routes use the /draft prefix so they won't conflict with the /:collectionId/:contentId pattern.

Actually since GET /draft/:token and GET /:collectionId/:contentId could conflict (both are GET with params), use the existing single router but ensure the /draft/:token route is matched first (defined before /:collectionId/:contentId). Hono matches in definition order, so define `/draft/:token` before `/:collectionId/:contentId`.

Mount the combined router at both `/api/preview` AND `/admin/preview` in app.ts.

2. **Update previewContent() in admin-content-form.template.ts:**

Replace the existing `previewContent()` function (line 915-936) that opens a new window with:

```javascript
function previewContent() {
  const form = document.getElementById('content-form');
  const collectionId = form.querySelector('[name="collection_id"]').value;
  const contentId = form.querySelector('[name="id"]').value || 'new';
  window.location.href = '/admin/preview/' + collectionId + '/' + contentId;
}
```

This navigates same-tab to the preview page (per CONTEXT.md: "Same-tab navigation -- preview page replaces admin, 'Back to editor' button returns").
