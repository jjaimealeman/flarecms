  1. `VALID_TRANSITIONS` map:
     ```
     draft -> [published, archived]
     published -> [draft, archived]
     archived -> [draft]
     ```

  2. `validateStatusTransition(from: string, to: string): { valid: boolean; error?: string }` function:
     - If `from === to`, return `{ valid: true }` (no-op, not an error)
     - If transition is valid per map, return `{ valid: true }`
     - If transition is invalid, return `{ valid: false, error: "Cannot transition from '{from}' to '{to}'. Allowed: {list}" }`

  3. `isSlugLocked(content: { published_at: number | null }): boolean` function:
     - Returns true if `published_at` is not null (content was published at least once)
     - Slug lock means the slug field cannot be changed

  4. `getUnpublishUpdates(): Record<string, any>` helper that returns the fields to clear when unpublishing:
     - `published_at: null`
     - `scheduled_publish_at: null`
     - `scheduled_unpublish_at: null`

  Export all functions. Also add the export to `sonicjs-fork/packages/core/src/index.ts` (in the services section) so downstream apps can use it if needed.

  This service is the SINGLE source of truth for status logic. Do NOT duplicate transition logic in route handlers.
