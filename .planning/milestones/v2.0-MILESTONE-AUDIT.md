## Cross-Phase Integration

### Connected (working)

1. **Core -> CMS** -- `createFlareApp`, `registerCollections`, `SchedulerService` all imported and used
2. **Workflow routes in app.ts** -- `createWorkflowRoutes()` and `createWorkflowAdminRoutes()` mounted directly at `/api/workflow` and `/admin/workflow`
3. **Workflow sidebar** -- Nav item renders for editor+ roles, points to `/admin/workflow/dashboard`
4. **Content -> Workflow enrollment** -- `api-content-crud.ts` calls `WorkflowEngine.initializeContentWorkflow()` directly on create and status change
5. **Workflow DB tables** -- `005_stage7_workflow_automation.sql` in migrations-bundle creates all required tables
6. **Content Layer Loader -> Site** -- `flareLiveLoader` exported from `@flare-cms/astro`, wired in `live.config.ts` for 5 collections
7. **Docs nav chain** -- `getLiveCollection` -> `buildNavTree()` -> sidebar, breadcrumbs, prev/next
8. **Search end-to-end** -- Search index endpoint -> MiniSearch -> SearchModal in Layout.astro
9. **Live Preview end-to-end** -- Admin POST draft -> token -> site fetches via `getDraftContent()`
10. **Schema Migrations UI** -- Routes mounted at `/admin/schema-migrations`, calls `SchemaMigrationService`
11. **Auth protection** -- All sensitive routes check `requireAuth()` and role constraints
12. **Docs collections registered** -- Both `docsCollection` and `docsSectionsCollection` in CMS index

### Broken / Orphaned

1. **`schema_migrations` DDL missing from bundle** -- `010_schema_migrations.sql` exists but is not in `migrations-bundle.ts`. Fresh installs will 500 on `/admin/schema-migrations`.
2. **`workflowPlugin` is dead code** -- Never activated via PluginManager, never exported from `core-plugins/index.ts`. Hook names are also wrong (`content:create` vs `content:after-create`). The direct wiring in `app.ts` compensates.
3. **Workflow routes not in public API** -- `createWorkflowRoutes`/`createWorkflowAdminRoutes` not exported from `routes/index.ts` or `core/index.ts`, only usable via internal import path.

**Integration Score: 12/15 connections verified**

## Tech Debt by Phase

### Phase 9: Schema Migrations UI
- **`schema_migrations` table DDL not in migrations-bundle.ts** -- The SQL file exists at `010_schema_migrations.sql` but was never added between `009_system_logging.sql` and `011_config_managed_collections.sql`. This is the single most impactful debt item: the entire Schema Migrations UI returns 500 on fresh installs.

### Phase 10: Workflow Engine Activation
- **`workflowPlugin` object is dead code** -- Never passed to `config.plugins.instances`, never activated through PluginManager. All its `builder.addHook()`, `builder.addAdminPage()`, `builder.addMenuItem()` calls produce no effect.
- **Hook names mismatch** -- Plugin registers `content:create` but system fires `content:after-create`. Would fail even if activated.
- **Not exported from core-plugins** -- Other plugins (email, otp, turnstile, ai-search) are all exported; workflow is not.
- **Routes not in public API** -- `createWorkflowRoutes`/`createWorkflowAdminRoutes` only reachable via direct internal import, not through `@flare-cms/core` exports.
- **`/api/workflow/analytics`** -- Route exists but no UI page links to it.

### Phase 6: Search & Deploy
- **Missing VERIFICATION.md** -- Phase was never formally verified. All artifacts exist and work, but the verification step was skipped.
- **REQUIREMENTS.md stale** -- SITE-02 and DEPLOY-01 not ticked despite being complete.

### Total: 8 items across 3 phases

## Verdict

All 28 requirements are satisfied. All user-facing flows work except the Schema Migrations admin page on fresh installs (missing table). The workflow engine functions correctly via direct wiring -- the plugin abstraction layer is dead code that should either be cleaned up or properly activated. No blockers for end users; debt is primarily around code hygiene and a single missing migration entry.
