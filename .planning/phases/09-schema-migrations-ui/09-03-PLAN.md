  <files>packages/core/src/templates/pages/admin-schema-migrations-history.template.ts</files>
  <action>
Create the migration history page template that renders within the admin layout.

**Export:** `renderSchemaMigrationsHistoryPage(data: SchemaMigrationsPageData): string`

**Interface:**
```typescript
interface SchemaMigrationsPageData {
  migrations: Array<{
    id: string
    collectionId: string
    collectionName: string
    description: string
    changes: Array<{
      type: string
      fieldName: string
      fieldLabel?: string
      fieldType?: string
    }>
    status: string
    appliedBy: string | null
    appliedAt: number
    rolledBackAt: number | null
    rolledBackBy: string | null
  }>
  total: number
  page: number
  pageSize: number
  collectionFilter?: string
  collectionName?: string
  collections: Array<{ id: string, name: string, display_name: string }>
  user?: { name: string, email: string, role: string }
  version?: string
}
```

**Template structure:**
- Use `renderAdminLayoutCatalyst()` wrapper (same as other admin pages)
- Page title: "Schema Migrations" (or "Schema Migrations: {collectionName}" if filtered)
- Subtitle: "Track all field changes across your collections"

**Filter bar:**
- Collection dropdown to filter by collection (all collections listed)
- Uses standard `<select>` with `onchange="window.location.href=..."` to filter
- "All Collections" option that clears the filter

**Migration list:**
- Each migration is a card/row with:
  - Status badge: green "Applied", yellow "Rolled Back", red "Failed" -- use same badge pattern as `getFieldTypeBadge()` in collections form template
  - Human-readable description (bold, primary text)
  - Collection name (secondary text)
  - Relative timestamp ("2 hours ago", "3 days ago") -- compute in template from appliedAt using a simple relative time function
  - User who applied (email or "System")
- Expandable details section (use `<details><summary>` HTML element for no-JS expand):
  - Changes list: each change shows type (Added/Modified/Removed), field name, field type
  - If rolled back: show rollback timestamp and user
- Empty state: "No schema migrations recorded yet. Changes will appear here when you add, edit, or remove collection fields."

**Pagination:**
- Show "Showing X-Y of Z migrations"
- Previous/Next links using `?page=N` query param
- Preserve collection filter in pagination links

**Styling:** Match the existing admin dark theme (zinc-800/900 backgrounds, white text, Catalyst design system classes). Use the same table/card patterns as the collections list page.
