Introduction

FlareCMS is a lightweight, edge-native headless CMS built specifically for the Cloudflare Workers ecosystem. It leverages global high-performance storage and compute to deliver content with sub-millisecond latency.

Why FlareCMS?

Traditional CMS architectures rely on centralized servers and databases, creating bottlenecks for global users. FlareCMS distributes your administrative UI and your content API across hundreds of edge locations simultaneously.

  • check_circle Edge-Native: Runs entirely on Cloudflare Workers, D1, and R2.
  • check_circle Zero Maintenance: Serverless infrastructure means no patching or scaling concerns.
  • check_circle Developer Experience: Git-based workflows and Type-safe API generation.

Next Steps

To begin using FlareCMS, you can initialize a new project using our CLI tool. This will scaffold a new worker with pre-configured routes and a basic admin UI.

src/index.ts

// Initialize your Flare application

import { createFlareApp } from '@flarecms/core';

const app = createFlareApp({

database: env.D1_DB,

storage: env.R2_BUCKET,

auth: { provider: 'cloudflare-access' }

});

export default app;
