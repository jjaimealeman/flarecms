```sql
CREATE TABLE IF NOT EXISTS schema_migrations (
  id TEXT PRIMARY KEY,
  collection_id TEXT NOT NULL,
  collection_name TEXT NOT NULL,
  changes TEXT NOT NULL,
  description TEXT NOT NULL,
  sql_executed TEXT,
  status TEXT NOT NULL DEFAULT 'applied',
  previous_schema TEXT,
  applied_by TEXT,
  applied_at INTEGER NOT NULL,
  rolled_back_at INTEGER,
  rolled_back_by TEXT
);

CREATE INDEX IF NOT EXISTS idx_schema_migrations_collection ON schema_migrations(collection_id);
CREATE INDEX IF NOT EXISTS idx_schema_migrations_applied_at ON schema_migrations(applied_at DESC);
CREATE INDEX IF NOT EXISTS idx_schema_migrations_status ON schema_migrations(status);
```

Note: No REFERENCES constraints on collection_id or applied_by -- keeps it simple and avoids FK issues if a collection or user is deleted after migrations are recorded. The collection_name is denormalized for display purposes.

**Types:**

```typescript
interface SchemaChange {
  type: 'add_field' | 'modify_field' | 'remove_field'
  fieldName: string
  fieldLabel?: string
  fieldType?: string
  previousConfig?: Record<string, any>
  newConfig?: Record<string, any>
}

interface SchemaMigration {
  id: string
  collectionId: string
  collectionName: string
  changes: SchemaChange[]
  description: string
  sqlExecuted: string | null
  status: 'applied' | 'failed' | 'rolled_back'
  previousSchema: string | null
  appliedBy: string | null
  appliedAt: number
  rolledBackAt: number | null
  rolledBackBy: string | null
}
```

**Class: SchemaMigrationService**

Constructor takes `db: D1Database`.

Methods:

1. `generateDescription(changes: SchemaChange[], collectionDisplayName: string): string`
   - Produces human-readable summary. Examples:
     - Single add: `"Added 'tags' field to Blog Posts"`
     - Single modify: `"Modified 'title' field in Blog Posts"`
     - Single remove: `"Removed 'excerpt' field from Blog Posts"`
     - Multiple: `"Added 'tags', modified 'title' in Blog Posts"`
   - Use field label if available, fall back to field name.

2. `async recordMigration(params: { collectionId: string, collectionName: string, changes: SchemaChange[], previousSchema: Record<string, any>, userId: string | null }): Promise<SchemaMigration>`
   - Generates a UUID for the migration id
   - Calls `generateDescription()` for the description
   - Stores `previousSchema` as JSON string
   - Stores `changes` as JSON string
   - Sets `status: 'applied'`, `appliedAt: Date.now()`
   - Inserts into `schema_migrations` table via `db.prepare().bind().run()`
   - Returns the full SchemaMigration object

3. `async getMigrationHistory(params: { collectionId?: string, limit?: number, offset?: number }): Promise<{ migrations: SchemaMigration[], total: number }>`
   - If collectionId provided, filters by it. Otherwise returns all migrations.
   - Orders by `applied_at DESC`
   - Default limit: 20
   - Returns total count for pagination
   - Parses `changes` JSON string back to SchemaChange[] for each result

4. `async getLastMigration(collectionId: string): Promise<SchemaMigration | null>`
   - Returns the most recent migration for a collection
   - Used for rollback (Plan 04)

5. `async updateMigrationStatus(migrationId: string, status: 'rolled_back', userId: string | null): Promise<void>`
   - Updates status and sets rolled_back_at / rolled_back_by
   - Used for rollback (Plan 04)

**Validation helper (static):**

6. `static validateFieldChange(params: { managed: boolean, fieldName: string, existingFields: string[] }): { valid: boolean, error?: string }`
   - Rejects if collection is `managed: true` (error: "Cannot modify fields on managed collections")
   - Rejects if fieldName is a reserved system field: `id`, `slug`, `title`, `status`, `created_at`, `updated_at`, `author_id`, `collection_id` (error: "'{fieldName}' is a reserved system field")
   - For add operations: rejects if fieldName already exists in existingFields (error: "Field '{fieldName}' already exists")

Export the class and types. Use `crypto.randomUUID()` for ID generation (available in Workers runtime).
