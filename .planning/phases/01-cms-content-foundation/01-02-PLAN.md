**1. Add a custom "Code Block" toolbar button:**

In the `toolbarButtons` array for the 'full' toolbar config, add a custom button object that:
- Has name `'code-block'`, className `'fa fa-code'`, title `'Insert Code Block'`
- On action: inserts a fenced code block template at cursor position:
  ```
  \`\`\`ts
  // code here
  \`\`\`
  ```
- Uses `easyMDE.codemirror.replaceSelection()` to insert the template
- Places cursor on the `// code here` line

Add this button after the `'table'` item in the full toolbar, before the preview group. Use EasyMDE's custom button format:
```javascript
{
  name: 'code-block',
  action: function(editor) {
    const cm = editor.codemirror
    const cursor = cm.getCursor()
    const codeTemplate = '```ts\\n// code here\\n```'
    cm.replaceSelection(codeTemplate)
    // Position cursor on the code line
    cm.setCursor({ line: cursor.line + 1, ch: 0 })
    cm.focus()
  },
  className: 'fa fa-code',
  title: 'Insert Code Block'
}
```

**2. Add R2 image upload via imageUploadFunction:**

Add `imageUploadFunction` to the EasyMDE constructor options. This enables drag-and-drop and paste image support:

```javascript
imageUploadFunction: function(file, onSuccess, onError) {
  const formData = new FormData()
  formData.append('file', file)
  fetch('/api/v1/media/upload', {
    method: 'POST',
    body: formData,
    credentials: 'include'
  })
  .then(response => {
    if (!response.ok) throw new Error('Upload failed: ' + response.status)
    return response.json()
  })
  .then(data => {
    // The media upload endpoint returns the file URL
    const imageUrl = data.url || data.data?.url || data.file?.url
    if (imageUrl) {
      onSuccess(imageUrl)
    } else {
      onError('No URL in upload response')
    }
  })
  .catch(err => {
    console.error('Image upload error:', err)
    onError(err.message || 'Upload failed')
  })
}
```

Also add `imageAccept: 'image/png, image/jpeg, image/gif, image/svg+xml, image/webp'` to the constructor options.

**3. Enable side-by-side preview by default in the toolbar:**

Ensure `'side-by-side'` is in the full toolbar config (it already is). No change needed but verify it's present.

**4. Increase default min height for docs content:**

Keep all existing dark mode styles, HTMX reinitialization, and textarea sync code unchanged.
