**Create `packages/site/src/lib/markdown.ts`:**

Build the unified pipeline as a module-level singleton (NOT created per-request). Follow the exact pattern from 03-RESEARCH.md "Complete Markdown Rendering Module" section. Key details:

- Use `createJavaScriptRegexEngine()` from `shiki/engine/javascript` (NOT the default Oniguruma WASM engine — it fails on Cloudflare Workers)
- Dual themes: `catppuccin-mocha` (dark) and `catppuccin-latte` (light)
- `keepBackground: false` (CSS handles background)
- Plugin order MUST be: remarkParse -> remarkRehype (allowDangerousHtml) -> rehypeRaw -> rehypeCallouts (github theme) -> rehypePrettyCode -> rehypeSlug -> rehypeAutolinkHeadings (behavior: 'append', content: span with '#') -> rehypeExternalLinks (target _blank) -> rehypeStringify
- Export `renderMarkdown(markdown: string): Promise<RenderResult>` where `RenderResult = { html: string, headings: Array<{ id: string, text: string, level: number }> }`
- Include `extractHeadings(html)` helper that parses h2/h3 from rendered HTML via regex (see research Pattern 2)
- Include a custom rehype plugin `rehypeTabGroups` that detects consecutive `[data-rehype-pretty-code-figure]` elements where the code block had `tab="Label"` in the meta string. Wrap them in `<div class="tab-group" data-tab-group="auto-N">` with tab buttons. Implementation: walk the HAST tree, find consecutive figure siblings with `data-rehype-pretty-code-figure` where the child pre has a tab data attribute from meta string parsing. Group them, create wrapper div with button bar and panels. The `data-tab-group` category should be derived from the tab labels (e.g., if tabs are "TypeScript" and "JavaScript", category is "language"; if "npm" and "pnpm", category is "package-manager"; otherwise "auto-N" with incrementing counter).

**Create `packages/site/src/styles/code-blocks.css`:**

Follow the exact CSS from 03-RESEARCH.md "Dual Theme CSS for Code Blocks" section. Include:
- `[data-rehype-pretty-code-figure]` relative positioning + margin
- `[data-rehype-pretty-code-title]` styling (filename display)
- Pre element: overflow-x auto, padding, border-radius 0.5rem, border slate-700/50, background slate-900
- `[data-line]` horizontal padding
- `[data-highlighted-line]` indigo background + left border
- Language label via `[data-language]::before` pseudo-element (absolute top-right, uppercase, muted color)
- Dual theme media query: `@media (prefers-color-scheme: dark)` switches `--shiki-dark` vars
- Copy button base styles (positioned absolute top-right of pre, opacity 0, shows on figure hover) — the JS that creates these buttons comes in Plan 02
- Tab group container and tab button styles

**Create `packages/site/src/styles/callouts.css`:**

Import rehype-callouts github theme: `@import 'rehype-callouts/theme/github';`
Then override CSS custom properties to match the site's dark slate palette:
- Note/Info: cyan-400 icon/border, slate-800 background
- Warning: amber-400 icon/border, slate-800 background
- Tip: emerald-400 icon/border, slate-800 background
- Caution: flare-500 (red-ish) icon/border, slate-800 background
- Text color: slate-200

**Create `packages/site/src/styles/prose.css`:**

Enhance the Tailwind prose-invert defaults:
- Heading anchors: `.heading-anchor` hidden by default, visible on heading hover, styled as muted text with hover color
- External link indicator: `a[target="_blank"]::after` with a small external-link SVG icon (inline SVG data URI or unicode arrow)
- Inline code: slightly different background from code blocks, rounded, padding
