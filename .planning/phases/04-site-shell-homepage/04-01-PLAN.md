  <files>

  </files>
  <action>
    Create `src/pages/sitemap.xml.ts` as an Astro API route that generates XML dynamically.

    1. Export a `GET` APIRoute handler.
    2. Build siteUrl from `site` parameter (from Astro context) with fallback to `https://flare-site.pages.dev`. Strip trailing slash.
    3. Define static pages array: `/` (priority 1.0, weekly), `/docs` (0.9, weekly), `/blog` (0.8, weekly), `/terms` (0.2, yearly), `/privacy` (0.2, yearly), `/code-of-conduct` (0.2, yearly).
    4. Fetch docs data using `getDocsSections()` and `getDocsPages()` from `../../lib/flare` (use Promise.all).
    5. Build dynamic docs URLs by matching each doc's section ID to sections, constructing `/docs/${sectionSlug}/${pageSlug}` paths (priority 0.7, monthly).
    6. Concatenate static + dynamic URLs.
    7. Generate XML string with `<?xml version="1.0" encoding="UTF-8"?>` header and `<urlset>` with sitemaps.org namespace.
    8. Return Response with `Content-Type: application/xml` and `Cache-Control: public, max-age=3600`.

    Wrap the data fetching in try/catch — if CMS API is unavailable, return sitemap with just static pages (don't crash the endpoint).

    Follow the exact pattern from 04-RESEARCH.md Pattern 2.
