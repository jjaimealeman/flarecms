<task type="auto">
  <name>Task 2: Create server-side search index endpoint</name>
  <files>packages/site/src/pages/api/search-index.json.ts</files>
  <action>
    Create `packages/site/src/pages/api/search-index.json.ts` as an Astro SSR API route:

    1. Import `APIRoute` from astro, `MiniSearch` from minisearch, `getDocsPages` and `getDocsSections` from `../../lib/flare`, and `MINISEARCH_OPTIONS` and `stripMarkdown` from `../../lib/search-config`
    2. Export `GET: APIRoute` handler that:
       a. Fetches sections and docs pages in parallel with `Promise.all`
       b. Builds a `sectionMap` (Map from section ID to `{ name, slug }`)
       c. For each doc page, creates a main document with: id (doc.id), title (doc.data.title), content (stripMarkdown of doc.data.content), section name, sectionSlug, slug, empty headingId, empty headingText
       d. Also extracts h2/h3 headings from raw markdown using `/^(#{2,3})\s+(.+)$/gm` regex. For each heading, creates a sub-document with id `${doc.id}#${headingId}`, the page title, heading text as content, section info, page slug, and the heading ID + text. Generate headingId by lowercasing, replacing non-word chars with hyphens, and trimming leading/trailing hyphens.
       e. Creates MiniSearch instance with `MINISEARCH_OPTIONS`, adds all documents
       f. Returns `new Response(JSON.stringify(miniSearch))` with headers:
          - `Content-Type: application/json`
          - `Cache-Control: public, max-age=300, s-maxage=300`
    3. Also include blog posts and news articles in the index (context says "Index ALL CMS-managed content"). Import `getBlogPosts` and `getNewsArticles` from flare.ts. For blog posts: id=post.id, title=post.data.title, content=stripMarkdown(post.data.content), section="Blog", sectionSlug="blog", slug=post.data.slug (or post.slug), headingId="", headingText="". Similar for news with section="News", sectionSlug="news". Blog URLs should be `/blog/${slug}`, news `/news/${slug}` -- handle this by storing sectionSlug so the client can build the correct URL pattern.
    4. Wrap the entire handler in try/catch -- on error, return a 500 with `{ error: 'Failed to build search index' }` JSON.
  </action>
