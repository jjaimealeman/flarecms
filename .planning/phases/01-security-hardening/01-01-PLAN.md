    ```typescript
    const JWT_SECRET_HARDCODED_DEFAULT = 'your-super-secret-jwt-key-change-in-production'
    ```

    Inside the validateBindingsMiddleware function, AFTER the existing missing bindings check (after the `if (missing.length > 0)` block), add:

    ```typescript
    // JWT_SECRET assertion — block all requests if using hardcoded default
    const jwtSecret = (c.env as any).JWT_SECRET
    if (!jwtSecret || jwtSecret === JWT_SECRET_HARDCODED_DEFAULT) {
      console.error('[Startup] FATAL: JWT_SECRET is not set or is using the hardcoded default. Run: wrangler secret put JWT_SECRET')
      return c.json({
        error: 'Service unavailable: JWT_SECRET must be configured — see wrangler secret put JWT_SECRET'
      }, 500)
    }
    ```

    Update the existing CACHE_KV warning message to note rate limiting specifically:
    ```typescript
    if (!c.env.CACHE_KV) {
      console.warn('[Startup] CACHE_KV binding not configured — rate limiting disabled')
    }
    ```

    The middleware ordering in src/index.ts already places validateBindingsMiddleware in `middleware.beforeAuth`, which runs before any auth or content routes. No changes to index.ts needed.

    IMPORTANT: Use `(c.env as any).JWT_SECRET` for the type access since the Bindings type from the fork may not include JWT_SECRET yet. This is a runtime check — the env var comes from .dev.vars or wrangler secrets.
