  **Part A: Change export format in my-astro-cms/src/index.ts**

  Currently the file does `export default createSonicJSApp(config)` which only exposes a fetch handler. Workers cron triggers require a `scheduled` export on the default object.

  Replace the export with:
  ```typescript
  import { SchedulerService } from '@sonicjs-cms/core'

  const app = createSonicJSApp(config)

  export default {
    fetch: app.fetch.bind(app),
    async scheduled(
      controller: ScheduledController,
      env: any,
      ctx: ExecutionContext
    ) {
      const scheduler = new SchedulerService(env.DB)
      ctx.waitUntil(scheduler.processScheduledContent())
    },
  }
  ```

  Note: Use `app.fetch.bind(app)` to preserve `this` context. The `env` type uses `any` because Bindings may not include all fields — or use the `Bindings` type if it's importable and correct. Check what the SchedulerService constructor actually expects and match it.

  Keep all existing imports and config unchanged. Only modify the export at the bottom.

  **Part B: Add cron trigger to wrangler.toml**

  Add to `my-astro-cms/wrangler.toml`:
  ```toml
  [triggers]
  crons = ["* * * * *"]
  ```

  Place it after the existing bindings sections. This runs every minute (UTC).
