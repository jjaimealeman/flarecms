For the FIRST production deploy, use the SAME resources as development (same D1 database ID, same R2 bucket, same KV namespace) since this is a single-instance deploy for now. Per-client isolation comes from the provisioning script.
    3. Update `[env.production]` to include:
       - `name = "my-sonicjs-app-production"`
       - `vars = { ENVIRONMENT = "production", CORS_ORIGINS = "https://my-astro-site.pages.dev", WEBHOOK_URLS = "", WEBHOOK_SECRET = "" }`
         NOTE: Use "https://my-astro-site.pages.dev" as the CORS origin — this is the default Pages URL. The actual URL will be confirmed after Pages project creation in Plan 04. If Pages project already exists, use its URL.
       - `[[env.production.d1_databases]]` with binding=DB, database_name="my-astro-cms-db", database_id="a2fe8bde-3cb8-4c0b-8a66-a996c482e5a3" (same as dev for now)
       - `[[env.production.r2_buckets]]` with binding=MEDIA_BUCKET, bucket_name="my-astro-cms-media"
       - `[[env.production.kv_namespaces]]` with binding=CACHE_KV, id="7dacbabdf6aa4896b0c9b0bc7b9125fe"
       - `migrations_dir` in the d1_databases section: same as root (`"./node_modules/@sonicjs-cms/core/migrations"`)
    4. Keep the existing `[env.staging]` section unchanged.
    5. Keep the root config unchanged.

    **Part B: Create DEPLOY-RUNBOOK.md**

    Create `my-astro-cms/DEPLOY-RUNBOOK.md` documenting the exact deploy procedure. Include:

    **Pre-deploy checklist:**
    - [ ] All code changes committed
    - [ ] Fork rebuilt: `cd sonicjs-fork && PATH="$(pwd)/node_modules/.bin:$PATH" pnpm build`
    - [ ] CMS dependency updated: `cd my-astro-cms && pnpm install`
    - [ ] CORS_ORIGINS in [env.production] set to actual Pages URL

    **First-time deploy (in order):**
    1. Create Pages project first (to get URL): `cd my-astro-site && wrangler pages project create my-astro-site`
    2. Update CORS_ORIGINS in wrangler.toml with the Pages URL from step 1
    3. Apply D1 migrations: `cd my-astro-cms && wrangler d1 migrations apply DB --remote --env production`
    4. Set secrets: `wrangler secret put JWT_SECRET --env production` (must be a strong random value, NOT the dev default)
    5. Optionally: `wrangler secret put WEBHOOK_SECRET --env production`
    6. Build and deploy CMS: `PATH="$(pwd)/../sonicjs-fork/node_modules/.bin:$PATH" pnpm build && wrangler deploy --env production`
    7. Verify: `curl https://my-sonicjs-app-production.workers.dev/api/system/health`
    8. Seed admin user if needed (SQL INSERT via `wrangler d1 execute DB --remote --env production --command "INSERT INTO users..."`)

    **Subsequent deploys:**
    1. Rebuild fork if changed
    2. `pnpm install` in my-astro-cms
    3. Check for pending migrations: `wrangler d1 migrations list DB --remote --env production`
    4. Apply if any: `wrangler d1 migrations apply DB --remote --env production`
    5. Build and deploy: `PATH="$(pwd)/../sonicjs-fork/node_modules/.bin:$PATH" pnpm build && wrangler deploy --env production`
    6. Verify health check

    **Rollback:**
    - `wrangler rollback --env production` (reverts Worker code, NOT D1 schema)
    - If D1 schema change caused the issue, schema rollback must be done manually

    **Secrets reference:**
    | Secret | Required | How to set |
    |--------|----------|-----------|
    | JWT_SECRET | Yes | `wrangler secret put JWT_SECRET --env production` |
    | WEBHOOK_SECRET | No | `wrangler secret put WEBHOOK_SECRET --env production` |

    **Troubleshooting:**
    - 500 on content endpoints after deploy: migrations not applied (`wrangler d1 migrations apply DB --remote --env production`)
    - CORS errors in browser: CORS_ORIGINS doesn't match the frontend URL
    - `tsup: command not found`: PATH doesn't include sonicjs-fork/node_modules/.bin
    - Health check returns degraded: check specific service status in response JSON
